# Prome Engine · 产品策划书

> 中国制造业出海的 AI 销售基础设施 —— 用一套 AI 系统替代海外获客与初步销售转化的人工流程。

---

## 1. 产品定位

**Prome Engine 是面向中国出海制造企业的多租户 AI SaaS，专注"海外获客 → WhatsApp 接待 → 询盘资格化"全链路自动化。**

它不是孤立的 AI 客服、广告工具或 CRM，而是把这三件事拧在一起的端到端销售系统：

```
┌─ 出广告 ─┐    ┌─ 接询盘 ─┐    ┌─ 筛线索 ─┐
│  AI 投手  │ ── │ AI 外贸员 │ ── │ 销售跟进 │
└────────┘    └────────┘    └────────┘
   Ogilvy        Medici         LeadHub
```

**产品边界**：只做"广告 → 询盘 → 资格化线索"的前段闭环。报价合同、订单履约、售后等成交后流程**不在范围**，由企业沿用现有 ERP / CRM 处理。

---

## 2. 产品愿景

> **成为中国制造企业出海的 AI 销售基础设施。**

终态下，企业主只需要做三件事：

1. **上传**企业与产品信息（PDF / Excel / 文档）
2. **定义**销售规则（什么客户算高质量、什么场景转人工）
3. **启动**广告预算

之后系统自动闭环：

- 制定 Click-to-WhatsApp 广告策略 → 生成素材 → 投放到 Meta
- 接待入站询盘 → 多轮对话 → 抽取结构化线索
- 按规则过滤 → 高质量线索推送给销售跟进
- 持续学习企业知识与话术，越用越懂这家公司

**销售只接"已经被 AI 谈过、确定要买"的客户**，不再处理大量低质询盘。

## 3. 目标用户群体

### 3.1 核心客户

中国出海的中小制造企业，特别是：

- **品类**：工程机械、汽车零部件、电子产品、化工原料、纺织等 B2B 大宗品
- **市场**：东南亚 / 中东 / 非洲 / 拉美等新兴市场
- **询盘量级**：每天 10–500 条 WhatsApp 询盘
- **痛点画像**：销售团队 2–10 人，被海量低质询盘淹没；想做 Meta 广告但缺投手 / 缺素材生产能力

### 3.2 使用者角色

平台是**邀请制开通**，1 个企业 = 1 个账号（无团队多人协作概念）：

| 角色 | 谁 | 在做什么 |
|---|---|---|
| **企业主 / 销售负责人** | 直接登录使用 Prome Engine 的人 | 配置产品线、上传知识、审核 AI 决策、跟进高质量线索 |
| **平台 Founder** | Prome Engine 团队 | 邀请新企业、租户管理、平台运维 |

### 3.3 核心痛点

| 痛点 | 现状 | Prome Engine 解决方式 |
|---|---|---|
| 海外获客成本高 | 投了广告没专业投手，素材生产慢，CTWA 配置门槛高 | Ogilvy AI 投手对话式生成完整广告计划，一键投到 Meta |
| WhatsApp 接待忙不过来 | 客户问"价格"、"MOQ"、"运到 X 国多少天"销售要重复回答 | Medici AI 外贸员 24×7 自动应答，多语言、上下文连续 |
| 询盘质量参差 | 大量询价后失联、问错产品的客户挤占销售时间 | AI 自动评估询盘质量（BAD / GOOD / QUALIFY / PROOF），只把高质量线索推给销售 |
| 报价 / 物流知识散落 | 销售各自掌握，新人难上手 | 知识库统一沉淀，AI 调用同一套数据回答所有客户 |
| 投放数据看不清 | 广告投了不知道哪条带来真实订单 | 全链路归因：从广告 ID → 询盘 → 客户成交，看到每条广告的真实 ROI |

## 4. 用户价值 & 核心卖点

### 4.1 用户价值

**1. 重新定义销售人力配比**
原来：1 个销售带 30 个询盘，70% 时间花在低质客户上。
现在：AI 处理 100% 第一接触，销售只接 AI 评估为 QUALIFY / PROOF 的高意向客户。

**2. 降低 Meta 广告投放门槛**
不需要专职投手。用中文描述需求 → Ogilvy 生成完整广告计划 → 审核 → 一键 Launch 到 Meta，全程对话式完成。

**3. 24×7 多语言即时响应**
入站消息 2 秒内 AI 介入，自动检测客户语言（中英法阿西俄等），不会因为时差或下班丢失商机。

**4. 知识资产沉淀**
销售经验、产品资料、运费表、合规要求一次上传永久复用。新员工不需要培训，AI 替代了知识传承环节。

**5. 投产闭环可量化**
每条 Meta 广告 → 询盘 → 线索 → 销售跟进的完整归因链路，AI 日报 / 周报自动生成投产分析。

### 4.2 核心卖点（对外统一表达）

> **一套系统，替代「海外投手 + WhatsApp 客服 + 初级销售」三个岗位。**

三句话版本：

- **AI 投手**：用对话生成 Click-to-WhatsApp 广告 → 一键投到 Meta
- **AI 外贸员**：自动接待 WhatsApp 询盘 → 多轮谈判 → 抽取结构化线索
- **销售只接高质量客户**：低质询盘被 AI 过滤，销售只跟进 AI 评估 QUALIFY / PROOF 的客户

### 4.3 核心差异

| | Prome Engine | 通用 AI 客服 / WA 营销工具 | 传统外贸 CRM |
|---|---|---|---|
| 广告生产 | ✅ Ogilvy 对话式生成 + 一键投放 | ❌ | ❌ |
| 入站对话 | ✅ Medici 多轮 + 工具调用 + 知识检索 | ⚠️ 通用 RAG，不懂销售流程 | ❌ |
| 线索资格化 | ✅ 4 档质量评级 + 业务价值分级 + 路由决策 | ❌ | ⚠️ 销售手动打分 |
| 投放归因 | ✅ 广告 → 询盘 → 线索全链路 | ❌ | ❌ |
| 部署模式 | ✅ SaaS，邀请制开通 | SaaS / 自部署 | 自部署 |

## 5. 平台功能介绍 & 使用指南

Prome Engine 由 6 个互相协作的模块组成。新企业从开通到 AI 跑起来约 30 分钟，引导路径：

```
开通账号  →  连接 Meta  →  建产品线  →  上传知识  →  配通知  →  AI 自动跑
  §5.1        §5.2          §5.3         §5.4         §5.5         §5.6
```

### 5.1 开通账号（邀请制）

- 平台 Founder 在 `/admin/invitations` 生成邀请链接发给企业主
- 企业主用链接走 `/signup`，注册即创建租户、初始化引导进度
- 一个企业 = 一个账号 = 一个独立数据空间，多个企业相互看不到对方任何数据

### 5.2 连接 Meta（`/settings/meta-connection`）

**两步向导，全程在前端 Console 风格日志面板可见每步 Graph API 调用**：

| 步骤 | 用户操作 | 系统做什么 |
|---|---|---|
| 1. 粘 token + BM ID | 在 Meta Business Suite 生成 system_user_token，复制 BM ID 一起粘进来 | 验证 token 所属 App / 列出可选 WhatsApp 号码与广告账户 |
| 2. 勾选资源 | 选定要绑定的 WABA + 1 个广告账户 | 写库 + 订阅 webhook + token AES-256-GCM 加密落库 |

**自动智能过滤**：
- WhatsApp 号码：测试号 / RED 质量 / 未认证业务名 → 自动隐藏
- 广告账户：仅 ACTIVE 状态可选；DISABLED / CLOSED / 欠款 / 风险审查中 → 自动隐藏
- 跨租户独占：BM / WABA / 广告账户 / WhatsApp 号码任一时刻只能属于一个企业，重复绑定会被拒

### 5.3 产品线（`/product-lines`）

**产品线是 Prome Engine 的核心抽象**：每条产品线对应一个独立的销售场景，绑定一个 WhatsApp 号码、一套 system prompt、一份 lead 字段定义、一个独立知识库。

| 配置项 | 说明 |
|---|---|
| 名称 + 描述 | 例如「重型卡车出口」 |
| 绑定的 WhatsApp 号 | 来自 §5.2 已绑定的号码池，每个号只能绑一条产品线 |
| Lead 字段定义 | 自定义这条业务线想抽取的字段（除标准字段外）。例如 `载重吨位`、`变速箱型号` |
| 资格化规则 | 哪些字段齐全算 QUALIFY、哪些算 PROOF；什么场景必须转人工 |
| 系统 Prompt 增量 | 这条产品线特有的话术、禁忌、报价口径 |

一个企业可以有多条产品线（例如同时做「重型卡车」和「卡车配件」两条线），每条独立运行，互不串扰。

### 5.4 知识库（`/product-lines/[id]/knowledge-base`）

按产品线挂载，6 层分类：

| Layer | 装什么 | 例 |
|---|---|---|
| **company** | 公司基础信息 | 公司简介、付款条款、售后政策 |
| **product** | 产品与价格 | 产品目录 Excel（SKU / 型号 / 价格 / MOQ）、规格书 |
| **logistics** | 物流与交付 | 各港口运费、交期、可达国家 |
| **compliance** | 合规与认证 | 各国进口认证、关税税率 |
| **sales** | 销售话术与流程 | SOP、异议处理、折扣规则 |
| **competitive** | 竞品情报 | 竞品价格对比、差异化话术 |

**四种入库方式**：
- **文档上传**：PDF / Word / Excel / CSV / Markdown / 纯文本，系统自动解析 → 切分 → 翻译 → 向量化
- **结构化导入（Excel 模板）**：价格表 / 运费表直接按列名映射入库，跳过 LLM 抽取，每行入库即 `confidence='verified'`
- **对话式 Teach**：直接用自然语言告诉 AI「我们 50HP 拖拉机出口肯尼亚 FOB 价 6800 美元」，AI 抽取成结构化知识点
- **Q&A 直填**：销售脑里的隐性知识——"客户问 X 我们答 Y" 多种问法 + 一段标准答 + 适用条件，medici 在客户问对不上具体 topic 时优先匹配

**资产管理**：可上传产品图片 / 规格 PDF / 资质证书等可发送资产。新版资产带结构化标签（type / view / color / scenario / linked_skus）+ caption 语义索引——medici 调 `find_asset` 优先按 tag 精确命中，没 tag 才走语义兜底；只有 tag 命中的图直接发给客户，semantic 命中需先文字描述确认。

**6 个 typed tool**：medici 不直接读 KB 表，而是调 `lookup_product` / `quote_price` / `lookup_shipping` / `lookup_policy` / `find_asset` / `check_constraint`。每个工具返回明确的成功 / 失败结构（`not_found` / `missing_fields` / `needs_human` / `unknown`），杜绝"靠相似度评估"的猜测式答复。

**学习闭环**：
- **复核队列**：低置信抽取 / 冲突写入隔离队列，未审不入活跃库（人工 approve / reject）
- **知识盲区**：medici 答不上的问题按 question_signature 自动聚合，告诉运营该补什么
- **纠正建议**：销售在 LeadHub 改写 medici 回复 → 系统建议作为 Q&A snippet 录入，一键采纳

**KB 体检**：自动统计每层覆盖率，识别"过期文档"和"知识盲区"，提示运营补充。

### 5.5 通知（`/settings/notifications`）

每个企业自己粘**飞书自定义机器人 webhook URL**，绑定到自己内部的销售群。

- 高质量询盘（route = HUMAN_NOW）→ 自动推 Markdown 卡片到群，含客户信息 / 询盘要点 / AI 摘要
- 一键测试按钮验证 webhook 能否正常发送
- webhook URL AES-256-GCM 加密落库，平台运维也看不到明文

### 5.6 自动跑起来（接入完成后日常使用）

#### a) Ogilvy · AI 投手（`/ogilvy`）

ChatGPT 风格的对话界面：

```
你 ：我想给重型卡车在尼日利亚做一波广告，预算 200 USD/天
AI ：好的，先理解几个前提...（追问目标受众、卖点、素材偏好）
AI ：[生成完整 plan_json：受众定向 + 多套素材文案 + 投放结构]
你 ：把第二个素材的标题改成"质保 3 年"
AI ：[更新 plan，重新预览]
你 ：好，发布
AI ：[一键 stage & launch 到 Meta，返回 campaign_id]
```

支持：
- 上传 PDF / 网址作为素材参考
- AI 自动调 Meta Graph API 检索 ad account / page / WhatsApp 号
- 全程 SSE 流式回复，每步工具调用过程可见

#### b) Medici · AI 外贸员（自动后台运行，无需 UI 操作）

每条入站 WhatsApp 消息：

```
客户消息 → 队列聚合（2 秒窗口合并连续短消息）
        → Medici 一次 LLM 调用同时完成：
          · 回复客户（多语言、上下文连续）
          · 调用 KB 工具（vector / structured / hybrid 三种检索）
          · 抽取结构化 lead（按产品线的 lead_fields）
          · 评估询盘质量（BAD / GOOD / QUALIFY / PROOF）
          · 评估业务价值（LOW / AVERAGE / HIGH）
          · 决定路由（CONTINUE / HUMAN_NOW / FAQ_END）
        → 写库 + 发回复 + 高质量线索推飞书
```

**人工接管**：销售随时可在对话页切换"接管"，AI 暂停，恢复后继续。

#### c) LeadHub · 询盘工作台（`/leadhub`）

销售的主战场，看所有产品线汇总的 leads：

- 按质量 / 业务价值 / 产品线 / 时间过滤
- 单条 lead 可看完整对话历史、客户画像（AI 生成）、所有抽取的结构化字段
- 批量「已跟进」标记
- 同步到外部 SCM 系统（如有）

#### d) 数据看板与 AI 报告（`/analytics` / `/reports`）

**Analytics**：
- 广告 dashboard：账户级汇总、按 campaign 分组指标、creative 预览
- 询盘 dashboard：按产品线 / 时间段统计询盘量、质量分布

**Reports**：
- 日报 / 周报 / 月报由 cron 自动生成
- 内容包括关键指标变化、线索质量分布、值得关注趋势、AI 行动建议
- 可点 SSE 流式查看 AI 现场重新生成
