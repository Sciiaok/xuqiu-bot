# 素材生成失败模式 prompt 模板(v2.0)

> SKILL.md §7.2 引用本文件。每次调用 `generate_ad_creative` 时,
> AI 必须在 `product_description` 中**显式补齐**以下 4 类失败模式的约束。
> 这些是「工具内部约束不可靠」的项,实测中已经多次发生问题。

---

## 1. 失败模式清单

### 失败模式 1:车身/车牌出现中文字符

**问题描述**:
- 车身侧面贴的中文广告语
- 车牌上的中文字符(如「京」「沪」「粤」)
- 后窗的中文贴纸

**实测案例**:福田 Aumark S 素材重生 4 次,车牌中文字反复出现。

**Prompt 必须显式补齐**:

```
车身侧面、车牌处保持完全干净,无任何中文或数字字符。
若必须显示车牌,使用目标市场常见车牌格式(无中文)。
```

---

### 失败模式 2:货币显示人民币 ¥ / CNY / RMB

**问题描述**:目标市场是菲律宾/阿联酋等,但素材里出现 ¥ 符号或 RMB / CNY 字样。

**实测案例**:多个会话生成的图片上出现 ¥。

**Prompt 必须显式补齐**:

```
价格标签(如有)使用 {目标市场货币符号},例如:
- 菲律宾 PH → ₱(PHP)
- 阿联酋 AE → AED
- 越南 VN → ₫(VND)
- 哈萨克斯坦 KZ → ₸(KZT)
- 沙特 SA → SAR
严禁出现 ¥ / CNY / RMB / 人民币符号。
```

> 注意:v2.0 已收紧,**广告内容中尽量不出现具体价格数字**(让客服在 WhatsApp 对话中沟通)。
> 若策略上不出现价格,此约束自然不会触发,但仍建议在 prompt 中加一句"严禁出现 ¥ / RMB"作保险。

---

### 失败模式 3:透视/比例不协调

**问题描述**:
- 车辆与场景透视不一致(车辆是俯视,场景是平视)
- 车辆比例失真(过大/过小)
- 车辆角度不符合广告吸引力

**实测案例**:福田 Aumark S 透视不对,与环境不融合。

**Prompt 必须显式补齐**:

```
车辆按 3/4 前侧角度(front three-quarter view),
与场景透视保持一致,不要俯视(top-down),不要鸟瞰。
车辆占画面 40-60%,自然融入场景。
```

---

### 失败模式 4:画面留白(角落空白)

**问题描述**:画面四角(尤其左上角)有大块未填充的纯色或白色空白。

**实测案例**:多个素材左上角未填满。

**Prompt 必须显式补齐**:

```
画面满铺,无空白角落,背景延伸到画面边缘。
若有装饰元素或文字,自然分布,不要在四角留大块空白。
```

---

## 2. Prompt 模板示例

完整的 `product_description` 示例(综合 4 类约束):

```
{车型} 行驶在 {目标市场} 的 {场景描述},
车辆按 3/4 前侧角度,与场景透视一致,不要俯视,
车辆占画面 40-60%,自然融入场景。

车身侧面、车牌处保持完全干净,无任何中文或数字字符。
若必须显示车牌,使用 {目标市场} 常见车牌格式。

画面满铺,无空白角落,背景延伸到画面边缘。

色调 {调性描述},氛围 {氛围描述}。
严禁出现 ¥ / CNY / RMB / 人民币符号。
```
