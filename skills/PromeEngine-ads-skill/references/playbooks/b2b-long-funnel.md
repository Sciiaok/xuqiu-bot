## 一、6 节点漏斗结构

```
广告曝光 → 点击 → {lead_channel}(询现货/OEM)
       → 高价值判断(车型 + 数量 + 目的港)
       → 报价单 / 框架协议
       → 合作(发合同)
       → 成交
```

`{lead_channel}` 按阶段 2 锁定的 `ad_format` 替换:
- CTW → **WhatsApp 对话**
- Lead Form → **表单提交**(字段:车型/数量/港口/电话)
- Web Conv → **着陆页转化**(Pixel + 表单)
- Google UAC → 应用 / 站内行为

漏斗后续节点(高价值判断/报价/合作/成交)对所有路径**通用**——只是入口渠道不同,B2B 长链路本身不变。

## 二、B2B 长链路核心特征(投放同事提炼)

### 2.1 三个关键特征

1. **转化周期极长**:30-60 天(从广告点击到实际成交),**不可能像电商实时看 ROI**
2. **单线索价值极高**:批量采购数万美元,**CPQL $30-50 完全可以接受**
3. **询盘质量差异巨大**:"I want to buy 50 Foton trucks to Iraq"(价值万美元) vs "Hi, what is the price?"(几乎零价值)

### 2.2 这些特征对 skill 输出的影响

- ROI 测算**不能用电商口径**(立即转化率),要用"30-60 天延后转化 + Qualified Lead 价值"模型
- KPI 监控**核心指标不是 ROAS**,是 **Cost per Qualified Lead(CPQL)**
- 受众/创意要**主动过滤低价值询盘**(在 welcome 阶段强制问公司/数量/港口)

## 三、客服响应 SLA

### 3.1 推荐 SLA(基于行业标准 + 投放同事经验)

| 时间窗口 | 行业建议 | 您当前实测 |
|---|---|---|
| **AI 首响** | ≤ 1 分钟 | ✅ 中位 0 秒(n=1247)|
| **人工首响** | ≤ 2 小时(工作时段) | ❌ 中位 12.1 小时 / P90 59.2 小时(n=46)|
| **报价回复** | ≤ 24 小时 | (待团队填) |
| **样车/合同跟进** | ≤ 48 小时 | (待团队填) |
| **周末/非工作时段** | 自动回复 + 工作日跟进 | (待团队填) |

### 3.2 您当前的瓶颈与建议

- **AI 首响完胜行业**,保持
- **人工接力 SLA 严重超标 6 倍**(行业 2h vs 您 12.1h)
- 建议:增值班人手 + 通知机制 + 自动提醒,把人工首响压到 ≤ 2 小时
- **AI 转人工率 7.1%**(正向),保留该机制,不要试图把 AI 处理率拉到 100%

### 3.3 客服配置(您当前)

- AI 首响 + 人工接力
- 客服语种:全语种支持

### 3.4 客服话术(按业务模式分支)

#### b2b-spot 对话脚本骨架

| 时段 | 销售动作 | 目标 |
|---|---|---|
| T+0(welcome) | 系统自动 | 留住经销商,兑现"现货"承诺 |
| T+2h(用户回复后) | 销售人工首响 | 建立信任,询问目标市场+采购规模 |

| T+48h | 跟进询问采购计划 + 样车安排 | 推进 |
| T+7d(若未签约) | 跟进询问障碍 + 提供促销条件 | 二次激活 |

#### b2b-general 对话脚本骨架

(转化周期可能 60-120 天,SLA 可放宽)

| 时段 | 销售动作 | 目标 |
|---|---|---|
| T+0 | AI welcome,引导提供公司信息 + 长期采购计划 | 资质判定 |
| T+24h | 提供 OEM 协作方案 + 长期合作要点 | 探索意向深度 |

## 四、高价值客户判定标准

### 4.1 量级标准(团队拍板)

- **单次至少采购**:>5 台
- **年累计意向采购**:>100 台
- **必须命中 4 项**(全部命中才算高价值):
  1. 提供公司信息
  2. 询问报价
  3. 提供具体采购数量
  4. 了解详细车辆信息

(此部分还需要补充,暂定 4 项)

### 4.2 客户身份偏好

**优先客户**(打勾代表优先服务):
- ✅ 经销商
- ✅ 进口商
- ✅ 车队公司
- ✅ 贸易商
- ⏸️ 其他身份(待 20 个高价值询盘多标"未识别",建议在 welcome 阶段强制问"您的身份是?")

**不接受**(归低质询单):
- 个人消费者
- 散客小批量

### 4.3 地理标准

**优先国家**(基于您 20 天实测综合表现 + 业务团队判断):
- 第一梯队:IQ / NG / SA / MX / PE
- 第二梯队:UZ / KZ / JO / EG / TN

**不接受国家**(政策/外汇/制裁原因):
- (待团队填,后续需要配置)

### 4.4 其他判定条件

**实测发现的隐性标准建议**:
- 必须提供 destination_country + car_model 至少 2 项(否则 58% 对话无法定性)
- qty > 100 台触发人工复核(防 NG 出现的 200 万台异常)
- 同 IP 多账号 / 同电话不同身份 → 风控复核

## 五、CPQL 上限决策

### 5.1 计算逻辑

```
1 个 Qualified Lead 的期望价值 = 单单总毛利 × Qualified Lead → 成交率
可承受 CPQL 上限 = 期望价值 × 10-30%
```

### 5.2 您的实测值

| 计算项 | 数值 | 备注 |
|---|---|---|
| 单台 FOB | 待团队填 | 06 文件 |
| 单台毛利率 | 待团队填 | 06 文件 |
| 单台毛利 | 待计算 | -- |
| 平均一单几台 | 待团队填(实测询盘中位 4 台,成交单未知) | 06 文件 |
| 单单总毛利 | 待计算 | -- |
| QL → 成交率 | 待团队填 | 06 文件 |
| 期望 1 QL 价值 | 待计算 | -- |
| **可承受 CPQL 上限** | **待团队填** | 06 文件 |

### 5.3 实测 CPQL 对照

- **您当前 CPQL = $14.49(严口径)** / **$2.17(宽口径)**
- 投放同事建议 $30-50(行业经验,非您实际)
- **结论**:即使按严口径,您 CPQL 还有 **2-3× 优化空间**,可以加预算扩量

### 5.4 加/缩预算触发规则(skill 阶段 4 输出建议)

**加预算触发**(任一满足):
- 实测 CPQL < 您拍板的上限 × 50%
- 实测 CTR > 3.0%
- 实测 Conversation → Qualified Lead > 30%

**缩/停预算触发**(任一满足):
- 实测 CPQL > 您拍板的上限
- 实测 CTR < 1.0% 持续 7 天
- 实测高质询单占比 < 5%(全是低质)
