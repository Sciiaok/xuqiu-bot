## 三、知识库维护规则

### 3.1 谁来维护

- **核心维护**:产品(用户自己)
- **专家审核**:投放同事
- **数据回填**:业务团队 + 投放团队

### 3.2 更新频率

- **每两周一次**(用户决策)

### 3.3 更新流程

```
1. 业务方/投放专家写 md 文件(按 _template-industry.md 模板)
   ↓
2. 提交到 Git 仓库
   ↓
3. 产品 review
   ↓
4. Git 合并 → CI 自动打包成 zip → 推到宿主目录
   ↓
5. 宿主自动检测 mtime 热加载(无需重启)
```

整个流程**不超过 5 分钟**,不需要后端,只需要 Git 写权限。

## 四、skill 启动时的自检流程

每次 skill 启动,做以下检查(AI 内部,不展示):

1. 读本文件,加载组合状态映射
2. 用户在阶段 1 提供 dim5 后,查表确认状态
3. 如果状态是"⚠️ 内测期·未验证",在产出末尾加标记
4. 如果某关键文件 mtime 比上次更新晚 >2 周,在会话开头主动提示用户:"距您上次更新知识库 已 X 天,是否回填新数据?"
