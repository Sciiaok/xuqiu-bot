# kb-usage-rules

本文件定义 AI 接待在谈单过程中如何使用知识库。

## 1. 总原则

- 知识库是标准售前问题的最高优先级依据
- 高风险信息必须优先依赖知识库
- 知识库没有有效结果时，不得编造答案

## 2. 必须优先查知识库的问题

### 2.1 价格

- 客户问价格、报价、FOB / CIF、最终价前置条件时，必须先查知识库
- 默认优先级建议为：优先 `calculate_price`，若条件不足或需解释规则，再补充 `search_knowledge`
- 该优先级在实现时需要结合研发最终的知识库存储方案设计，是否完全沿用该默认顺序，由研发结合现网知识库结构最终拍板

### 2.2 库存

- 客户问是否有货、是否现货、是否可供、能否马上发运时，必须先查知识库
- 若库存不足，推进替代颜色、替代配置、等待周期或分批安排

### 2.3 付款条款

- 客户问付款方式、账期、定金比例、尾款安排时，必须先查知识库
- 非标准付款条款不得自行承诺

### 2.4 船期 / 运费

- 客户问目的港运费、CIF 价格、船期、交付时效时，必须先查知识库
- 若目的港不明确，应先补齐目的港

### 2.5 公司资质 / 出口能力

- 客户问公司背景、资质、出口经验、市场出口可行性时，必须先查知识库

### 2.6 政策 / 责任 / 赔偿口径

- 客户问出口政策、责任归属、赔偿规则、法务边界时，必须先查知识库
- 即使查到标准口径，也不能超出标准内容进一步承诺

## 3. 知识库查不到时的处理

允许的动作：

- 明确告诉客户当前信息需要进一步确认
- 说明后续可由销售团队或人工继续跟进

禁止的动作：

- 编造价格
- 编造库存
- 编造船期
- 编造特殊付款条款
- 编造赔偿或责任承诺

## 4. 必须直接转人工的知识库异常场景

前提：报价所需关键字段已经收集齐全。当前整车业务线下，正式报价前必须至少补齐以下字段：

- 配置
- 颜色
- 数量
- 目的国
- 目的港
- 贸易方式
- 公司名称

在上述字段已补齐的前提下，当知识库出现以下情况时，应优先转人工：

- 无结果且客户要求正式报价
- 无结果且客户要求最终成交价
- 无结果且客户要求发 PI / 合同
- 结果冲突
- 结果明显过期
- 当前问题属于管理判断范围

若上述字段尚未补齐，则应优先继续补齐字段，而不是直接进入正式报价或最终成交判断。
