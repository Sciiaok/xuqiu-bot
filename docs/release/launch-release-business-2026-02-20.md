# Lead Engine 上线说明（业务版 / 非技术）

更新时间：2026-02-20  
版本基线：`main@d09c705`

## 1. 上线目标
本次上线目标是让“客户询盘 -> 线索识别 -> 销售跟进 -> 外部同步”这条链路更稳定、更快、更可追踪。
1. 提升客服响应效率与会话处理稳定性。
2. 提升销售获取高价值线索的速度与准确率。
3. 提升运营批量处理（审批/同步）效率。
4. 提升管理层对线索状态与转化过程的可见性。

## 2. 整体工作流（业务视角）
1. 客户通过 WhatsApp 发送文字或语音消息。
2. 系统自动接收并处理消息（语音会转文字）。
3. AI 理解会话内容，提取询盘信息并判断线索质量。
4. 系统根据结果执行路线：继续跟进、转人工销售、或 FAQ 收口。
5. 客服/销售可在后台查看会话与线索，进行编辑、审批、同步。
6. 已审批线索可同步到外部系统，状态全程可追踪。

## 3. 对业务的价值
1. 客服价值：Inbox 会话处理链路更清晰，提升跟进效率。
2. 销售价值：更快拿到可跟进线索，减少漏单和无效沟通。
3. 运营价值：支持批量审批和批量同步，减少重复操作。
4. 管理价值：线索质量、流转路径、同步状态更透明。

## 4. AI 价值与当前能力

### 4.1 当前 AI 能力怎么运用
1. 当前模式是“单次综合判断”：一次 Claude 调用同时完成意图识别、线索抽取、质量分级、路由建议、回复文案生成。
2. 优点是上线快、链路短、维护简单。
3. 当前限制是不同能力耦合较高：某一块策略变化时，通常要整体改 Prompt 和整体回归测试。

### 4.2 当前业务侧可感知能力
1. 自动识别询盘意图与对话方向。
2. 自动提取车型、目的地、数量、贸易条款等关键信息。
3. 自动给出线索质量与业务价值判断。
4. 自动执行分流策略（继续跟进、转人工、FAQ 收口）。
5. 支持一个会话中的多线索场景。

## 5. AI 未来方向（重点）

### 5.1 按职能拆分为多个 API 调用
建议把“会话理解 -> 输出执行”拆为多个能力模块，每个模块职责单一、可独立迭代：
1. 意图识别模块：识别客户是询价、合作咨询、个人购车、无效消息等。
2. 信息抽取模块：抽取车型、目的地、数量、贸易条款、公司名等结构化字段。
3. 质量评分模块：输出 询盘质量，商业价值，优先级。
4. 回复生成模块：根据当前目标生成对客话术。
5. 路由决策模块：决定 继续AI对话 / 人工处理 / 对话结束

### 5.2 后台配置化 SKILL 的核心想法
目标是把“能力策略”从代码中抽离，让运营/产品可配置：
1. SKILL 作为可配置能力单元：定义输入、输出、Prompt 模板、模型、温度、超时、重试策略。
2. 后台可配置调用编排：按场景组合调用顺序（例如先抽取、再评分、再生成回复）。
3. 支持版本管理：每个 SKILL 有版本号，可灰度发布与快速回滚。
4. 支持 A/B 测试：同场景可对比不同 SKILL 版本效果。

### 5.3 这条路线对业务的直接收益
1. 迭代更快：调“抽取规则”不必影响“回复话术”。
2. 风险更低：单模块变更可控，回归范围更小。
3. 更可运营：业务可参与策略配置，不必每次都走发版。
4. 更可度量：每个模块都可单独看准确率、失败率、耗时与成本。

### 5.4 推荐落地节奏（分阶段）
1. 第一阶段：保持当前单调用主链路，同时新增“模块化影子链路”做对照，不直接影响线上回复。
2. 第二阶段：先切分“信息抽取模块”，其余模块仍用现有逻辑，验证收益。
3. 第三阶段：逐步拆分评分、路由、回复模块，引入后台 SKILL 配置与版本控制。
4. 第四阶段：实现全链路可编排，支持按客户类型/国家/产品线使用不同 SKILL 组合。

## 6. 后台功能变化（放在后面）

### 6.1 Leads 页面
1. 线索列表支持按质量/价值/客户/车型筛选。
2. 支持批量审批、批量同步。
3. 支持编辑线索关键信息（车型、目的地、贸易条款等）。
4. 可看到线索是否已成功同步到外部系统。

### 6.2 Inbox 页面
1. 默认打开后自动进入第一个联系人对话。
2. 只加载当前联系人消息，其他联系人点击后再加载。
3. 消息加载方式调整为按联系人逐步加载。
4. 客服发送消息后，系统会写入会话记录。

### 6.3 Contacts 页面
1. 可查看每个客户的线索数、会话数、消息数。
2. 支持从客户视图快速跳转去跟进会话。

## 7. 上线当天怎么确认“正常”
业务侧只需关注以下 8 项：
1. 可以正常登录后台。
2. Leads 页面可加载并筛选。
3. 可以编辑并保存线索。
4. 可以审批线索。
5. 可以触发同步，且同步状态更新。
6. Inbox 页面打开后可直接看到一个联系人会话。
7. 客户发送消息后，后台能看到消息并回复。
8. Contacts 页面统计信息可正常展示。

## 8. 风险与回滚（业务视角）

### 8.1 已知风险与应对
1. 风险：外部服务波动（WhatsApp、AI、外部同步平台）。
应对：系统有重试与日志，可快速定位并补偿处理。
2. 风险：高峰期消息积压。
应对：有队列处理与兜底定时任务，降低丢消息风险。
3. 风险：外部同步失败。
应对：失败会记录状态，可重试，不影响线索本地留存。

### 8.2 回滚原则
如出现影响核心业务的问题（无法收消息、无法回复、线索无法查询），可执行“版本回退到上一稳定版本”。
回退后优先保障：
1. 收消息与回消息可用。
2. Leads 查询可用。
3. 同步功能可暂时降级，但不丢线索。

## 9. 业务沟通口径
建议对销售与客服同步以下口径：
1. Inbox 默认自动打开一个联系人。
2. 线索支持更完整编辑与批量操作。
3. 同步状态更清晰，若失败可重试。
4. 如遇异常，先记录联系人 wa_id 与时间点，便于快速排查。
